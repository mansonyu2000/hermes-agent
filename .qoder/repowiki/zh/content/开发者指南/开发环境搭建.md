## 简介

Hermes Agent 是由 Nous Research 开发的自改进型 AI 助手，具备内置学习循环能力，能够从经验中创建技能、在使用过程中改进技能，并支持多平台部署。本指南专注于为开发者提供完整的本地开发环境搭建方案。

## 系统要求与前置条件

### 操作系统支持
- **Linux**: 完全支持
- **macOS**: 完全支持  
- **Android/Termux**: 部分功能支持
- **Windows**: 不支持（需使用 WSL2）

### 核心依赖要求

#### Python 环境
- **Python 版本**: 3.11+（推荐 3.11）
- **包管理器**: uv（首选）或 pip
- **虚拟环境**: 建议使用 Python 3.11 虚拟环境

#### 开发工具
- **Git**: 版本控制系统
- **Node.js**: 用于浏览器工具（版本 22 LTS）
- **系统包管理器**: 
  - Ubuntu/Debian: apt
  - Fedora: dnf  
  - Arch: pacman
  - macOS: brew
  - Android/Termux: pkg

#### 可选但推荐的系统工具
- **ripgrep (rg)**: 快速文件搜索
- **ffmpeg**: 语音消息处理
- **Docker**: 容器化运行（可选）

## 完整安装步骤

### 方法一：一键安装脚本（推荐）

```bash
# 使用官方安装脚本进行快速安装
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

安装脚本会自动完成以下操作：
1. 检测系统类型和架构
2. 安装/更新 uv 包管理器
3. 创建 Python 3.11 虚拟环境
4. 安装所有依赖包
5. 设置环境变量和路径
6. 同步内置技能到用户目录

### 方法二：手动克隆安装

```bash
# 1. 克隆仓库
git clone https://github.com/NousResearch/hermes-agent.git
cd hermes-agent

# 2. 安装 uv 包管理器
curl -LsSf https://astral.sh/uv/install.sh | sh

# 3. 创建虚拟环境
uv venv venv --python 3.11

# 4. 激活虚拟环境
source venv/bin/activate

# 5. 安装开发依赖
uv pip install -e ".[all,dev]"
```

### 方法三：使用 Nix 开发环境

```bash
# 进入 Nix 开发环境
nix develop

# 或者构建包
nix build
```

## CLI 配置详解

### 配置文件位置与优先级

Hermes Agent 支持两种配置文件：

1. **用户配置** (`~/.hermes/config.yaml`)
2. **项目配置** (`cli-config.yaml`)

配置加载顺序：
```
~/.hermes/config.yaml (优先)
↓
cli-config.yaml (备选)
↓
cli-config.yaml.example (默认值)
```

### 核心配置模块

#### 模型配置 (model)
- **provider**: 推理提供商选择
  - auto: 自动检测（默认）
  - openrouter: OpenRouter
  - anthropic: Anthropic API
  - custom: 自定义本地服务器
- **base_url**: API 基础地址
- **default**: 默认模型名称

#### 终端后端配置 (terminal)
支持多种执行环境：

1. **本地执行 (local)** - 默认
2. **SSH 远程执行 (ssh)**  
3. **Docker 容器 (docker)**
4. **Singularity 容器 (singularity)**
5. **Modal 云执行 (modal)**
6. **Daytona 云沙盒 (daytona)**

#### 安全扫描配置 (security)
- **tirith_enabled**: 启用命令安全扫描
- **tirith_path**: 扫描器路径
- **tirith_timeout**: 超时时间

#### 上下文压缩配置 (compression)
- **enabled**: 启用自动压缩
- **threshold**: 触发阈值（默认 50%）
- **target_ratio**: 保留比例（默认 20%）

## 常见问题与故障排除

### Python 版本问题

**问题**: Python 版本过低
**解决**:
```bash
# 使用 uv 安装指定版本
uv python install 3.11
uv venv venv --python 3.11
```

**问题**: 虚拟环境未激活
**解决**:
```bash
# 激活虚拟环境
source venv/bin/activate
# 或在 Nix 环境中
nix develop
```

### 依赖安装失败

**问题**: uv 安装失败
**解决**:
```bash
# 手动安装 uv
curl -LsSf https://astral.sh/uv/install.sh | sh
# 或使用包管理器
pip install uv
```

**问题**: 依赖冲突
**解决**:
```bash
# 清理并重新安装
rm -rf venv
uv venv venv --python 3.11
uv pip install -e ".[all]"
```

### Node.js 相关问题

**问题**: 浏览器工具不可用
**解决**:
```bash
# 安装 Node.js 22 LTS
# 在 Linux/macOS 上
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs

# 在 Android/Termux 上
pkg install nodejs

# 安装 agent-browser
npm install -g agent-browser
agent-browser install
```

### 权限和路径问题

**问题**: hermes 命令找不到
**解决**:
```bash
# 检查符号链接
ls -la ~/.local/bin/hermes
# 重新创建链接
ln -sf $(pwd)/venv/bin/hermes ~/.local/bin/hermes
```

**问题**: Docker 权限不足
**解决**:
```bash
# 添加用户到 docker 组
sudo usermod -aG docker $USER
# 重新登录或执行
newgrp docker
```

### 网络和 API 配置

**问题**: API 认证失败
**解决**:
```bash
# 检查 .env 文件
cat ~/.hermes/.env
# 验证 API 密钥格式
hermes doctor
```

**问题**: 网络连接超时
**解决**:
```bash
# 检查防火墙设置
sudo ufw status
# 配置代理（如果需要）
export HTTP_PROXY=http://proxy.example.com:8080
export HTTPS_PROXY=https://proxy.example.com:8080
```

## 性能优化建议

### 依赖管理优化

1. **使用 uv 进行依赖管理**
   - 更快的安装速度
   - 更好的缓存机制
   - 锁定文件支持

2. **选择合适的依赖变体**
   ```bash
   # 开发环境
   uv pip install -e ".[dev]"
   # 生产环境
   uv pip install -e ".[all]"
   # 最小化安装
   uv pip install -e "."
   ```

### 系统资源优化

1. **内存和磁盘配置**
   - Docker 容器: 5GB 内存, 50GB 磁盘
   - SSH 后端: 根据目标服务器配置
   - 本地执行: 使用系统默认设置

2. **并发和批处理**
   - 调整工具调用轮次限制
   - 配置适当的超时时间
   - 启用适当的日志级别

### 缓存和持久化

1. **启用上下文压缩**
   - 自动减少长对话的令牌使用
   - 保护最近的对话内容
   - 配置合适的阈值和比例

2. **优化文件搜索**
   - 安装 ripgrep 提升搜索性能
   - 配置适当的索引策略
   - 使用工作树隔离避免文件冲突

## 总结

Hermes Agent 的开发环境搭建相对简单，主要依赖于现代 Python 生态系统的工具链。通过官方安装脚本可以快速完成环境配置，而手动安装方式则提供了更多的控制权和灵活性。

### 关键要点

1. **前置条件满足**: 确保 Python 3.11+、Git、Node.js 22 正确安装
2. **包管理器选择**: 优先使用 uv，它提供了更好的性能和用户体验
3. **配置文件管理**: 合理使用 `~/.hermes/config.yaml` 和 `cli-config.yaml`
4. **环境验证**: 使用 `hermes doctor` 进行全面的环境诊断
5. **故障排除**: 利用内置的诊断工具和错误提示信息

### 下一步行动

1. 完成基础环境搭建
2. 运行 `hermes doctor` 验证环境
3. 执行 `hermes setup` 进行初始配置
4. 尝试基本的聊天功能
5. 探索高级功能和配置选项

通过遵循本指南，开发者可以快速建立一个功能完整的 Hermes Agent 开发环境，为后续的功能开发和调试工作奠定坚实的基础。
